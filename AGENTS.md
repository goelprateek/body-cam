# AGENTS

## Purpose
This repository hosts the `body-cam` project and related implementation guides for backend, frontend, mobile, infrastructure, and architecture work.

## Working Agreement
- Keep changes scoped to the feature or bug being worked on.
- Prefer documenting decisions in `PLANS.md` before broad architectural changes.
- Use the local skills in `.codex/skills/` to stay consistent across modules.

## Project Areas
- `backend/` for Spring Boot services when added.
- `frontend/` for Angular applications when added.
- `android/` for Android applications when added.
- `deploy/` or `infra/` for Docker and environment setup when added.

## Definition Of Done
- Code builds in the affected module.
- Config and docs are updated together.
- Cross-layer impacts are called out explicitly.
